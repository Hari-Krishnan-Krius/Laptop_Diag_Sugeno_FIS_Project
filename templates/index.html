🤖 Agent Setup — Monitor Any Laptop

Install the agent on each laptop you want to monitor. No pre-registration needed — the laptop appears in the dashboard automatically when the agent first connects.

Step 1 — Install dependencies
pip install psutil requests
# Windows only (for real voltages & fan speed):
pip install wmi pywin32
# Also run Libre Hardware Monitor as Administrator
Step 2 — Copy the agent file

Copy agent/laptop_agent.py from the server package to the laptop (USB, network share, email — any way works).

Step 3 — Set environment variables
DIAG_SERVER_URL=http://<YOUR_SERVER_IP>:5000
DIAG_API_KEY=<AGENT_API_KEY from server .env>
Optional: DIAG_INTERVAL=600 (seconds, default 10 min) · DIAG_EMAIL=alerts@corp.com · DIAG_NAME="Lab PC 01"
Step 4 — Run the agent
python laptop_agent.py --test   # verify sensor readings
python laptop_agent.py --once   # send one report, check dashboard
python laptop_agent.py          # run continuously (every 10 min)
Auto-start — Linux (systemd)
sudo cp laptop_agent.service /etc/systemd/system/
# Edit the file — set User, WorkingDirectory, and env vars
sudo systemctl enable --now laptop_agent
sudo journalctl -fu laptop_agent    # live logs
Auto-start — Windows (Task Scheduler)
Task Scheduler → Create Task → Trigger: At startup → Action: python C:\DiagAgent\laptop_agent.py
Set DIAG_SERVER_URL and DIAG_API_KEY as System environment variables.
Scaling to 1000 laptops? Copy laptop_agent.py and set the two env vars on each machine. Every agent registers itself — no server-side setup per machine. Use a tool like Ansible, SCCM, or a simple login script to deploy the file and set the env vars at scale.
➕ Manual Registration

Use this for the server machine itself (is_local) or for pre-registering a laptop before the agent is installed. For all other cases, just run the agent — it registers automatically.
